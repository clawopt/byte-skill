# 第 1 章 · 走进计算机网络世界

计算机网络是让全球计算机互联互通的基础设施。从你打开浏览器访问一个网页，到手机 App 收发消息，都在依赖网络协议栈默默工作。

## 1.1 互联网是如何「连起来」的？

从抽象上看，一次网络访问大致会经历：

1. 客户端应用发起请求（如浏览器输入 URL 并回车）；
2. 通过 **TCP/IP 协议栈** 将数据切分、封装成报文；
3. 经过局域网、运营商骨干网、多级路由器转发；
4. 抵达目标服务器，再由服务器应用处理并返回响应。

在这个过程中，涉及到 IP、TCP、HTTP、DNS 等一系列协议协同工作。

## 1.2 分层模型的直观理解

学习网络时最常见的一个抽象是「分层模型」，例如：

- 应用层（HTTP、RPC、WebSocket 等）
- 传输层（TCP、UDP）
- 网络层（IP）
- 数据链路层（以太网、Wi-Fi）

分层的核心价值是：**每层只关心自己的职责**，对下层屏蔽细节，对上层提供清晰接口。

## 1.3 本教程的学习路径

在之后的章节中，你将从「整体视角」出发，拆解计算机网络的关键主题：

1. 从一次 HTTP 请求看完整网络传输路径；
2. TCP 连接、滑动窗口、拥塞控制等核心机制；
3. DNS、CDN、反向代理在互联网访问中的角色；
4. 常见网络问题的排查思路与抓包分析方法。

通过这些内容，你可以从「会用」逐步走向「看得懂、能排障」。
