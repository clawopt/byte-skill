# 第 1 章 · 操作系统是什么

操作系统（Operating System，OS）是连接 **硬件** 与 **应用程序** 的核心软件，它负责管理 CPU、内存、硬盘、网络等资源，并向上层提供统一、抽象的接口。

## 1.1 为什么需要操作系统？

如果没有操作系统，每个应用程序都需要自己处理：

- 如何把程序加载到内存并运行；
- 如何与磁盘、网卡、显示器交互；
- 如何与其他程序「和平共处」，避免互相覆盖内存。

这在现代复杂系统中几乎是不可能完成的任务。因此我们需要一个「中间层」来：

- 帮我们管理硬件资源；
- 为应用提供统一 API；
- 提供权限隔离与安全机制。

## 1.2 操作系统的核心职责

- **进程与线程管理**：创建、调度、销毁进程/线程，并在它们之间进行上下文切换。
- **内存管理**：虚拟内存、分页机制、内存分配与回收，避免不同进程互相干扰。
- **文件系统**：以文件/目录的形式组织磁盘数据，提供读写接口和权限控制。
- **设备管理与驱动**：屏蔽底层硬件细节，为键盘、鼠标、显示器、网卡等提供统一访问方式。
- **安全与权限控制**：用户/组、访问控制列表、系统调用权限等。

## 1.3 本教程的学习路径

在后续章节中，你将从「进程/线程」开始，逐步学习：

1. 进程与线程模型；
2. 内存管理与虚拟地址空间；
3. 文件系统与 I/O 子系统；
4. 同步原语与并发控制；
5. 常见操作系统实现（Linux 为主）的关键设计与实践。

本教程会尽量结合实际 Linux 命令与代码示例，帮助你把抽象概念落到真实系统行为上。
