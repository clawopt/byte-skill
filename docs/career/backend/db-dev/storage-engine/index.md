# 存储引擎 · 第 1 章 InnoDB 存储模型概览

存储引擎是数据库「如何在磁盘上组织与读写数据」的核心组件。以 MySQL 的 InnoDB 为例，它通过页（Page）、段（Segment）、表空间（Tablespace）等概念组织数据。

## 1.1 为什么要了解存储引擎？

从业务代码看，执行一条 `SELECT`/`UPDATE` 似乎非常简单，但在底层会涉及：

- 如何在磁盘文件中定位到对应的数据页；
- 如何通过缓冲池（Buffer Pool）缓存热点数据；
- 如何在刷盘与性能之间做权衡。

理解这些细节有助于你：

- 设计更合理的主键与索引；
- 解释为什么某些 SQL 会导致大量随机 I/O；
- 在遇到 IOPS 瓶颈时与 DBA 更好地协同排查。

## 1.2 InnoDB 存储结构的几个关键概念

- **页（Page）**：InnoDB 通常以 16KB 作为最小读写单位；
- **索引组织表（Clustered Index）**：数据按主键顺序存放在 B+ 树叶子节点中；
- **二级索引（Secondary Index）**：叶子节点存储主键值，需要回表；
- **Buffer Pool**：内存中的页缓存，决定了很多查询的真实延迟。
