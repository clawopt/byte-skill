# 数据库原理 · 第 1 章 事务与并发控制概览

在数据库开发工程师的日常工作中，「事务是否正确」「并发是否安全」是最核心的问题之一。本章作为数据库原理的开篇，将从宏观视角介绍这些概念。

## 1.1 事务 ACID 特性

一个事务（Transaction）通常被期望满足 ACID 四大特性：

- **原子性（Atomicity）**：事务中的操作要么全部成功，要么全部回滚。
- **一致性（Consistency）**：事务前后数据应满足预定义的约束（如外键、唯一约束、业务不变量）。
- **隔离性（Isolation）**：并发事务之间互不干扰，视觉上仿佛是串行执行。
- **持久性（Durability）**：事务一旦提交，其结果应被持久保存，即使系统宕机也能恢复。

后续章节会结合日志与存储引擎实现，解释数据库是如何「保证」这些特性的。

## 1.2 并发异常与隔离级别

在没有任何控制的情况下，并发事务会带来：

- 脏读（Dirty Read）
- 不可重复读（Non-repeatable Read）
- 幻读（Phantom Read）

典型的 SQL 标准隔离级别包括：

- READ UNCOMMITTED
- READ COMMITTED
- REPEATABLE READ
- SERIALIZABLE

不同数据库在实现这些隔离级别时可能不完全一致，数据库开发工程师需要了解目标系统（如 MySQL InnoDB）的具体行为。
