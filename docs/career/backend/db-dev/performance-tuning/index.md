# 性能调优与排障 · 第 1 章 关键指标与常见瓶颈

在复杂的线上系统中，数据库往往是性能瓶颈的核心之一。数据库开发工程师需要具备「看指标就能大致知道问题在哪」的能力。

## 1.1 需要关注的核心指标

- QPS/TPS：请求吞吐量；
- 平均/ P95/ P99 延迟：整体响应时延分布；
- 活跃连接数、连接队列长度；
- 锁等待时间与死锁次数；
- Buffer Pool 命中率、磁盘 I/O 利用率等。

在后续章节中，我们会结合常见监控面板与系统视图，帮助你建立从指标到问题定位的思维路径。
