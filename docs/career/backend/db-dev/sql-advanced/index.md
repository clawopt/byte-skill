# SQL 与索引优化 · 第 1 章 执行计划初识

当你执行一条复杂 SQL 时，数据库内部会生成一份「执行计划」（Explain Plan），描述它将如何访问数据、如何使用索引。

## 1.1 为什么要看执行计划？

对于数据库开发工程师来说，执行计划是判断 SQL 性能问题的第一手资料：

- 是否命中了预期索引；
- 是否出现了全表扫描或回表次数过多；
- 是否存在明显的排序、临时表、Hash Join/ Nested Loop 等操作。

在后续内容中，我们会以 MySQL 为例，逐步拆解 `EXPLAIN` 输出中的各个字段，并结合真实业务场景分析索引设计的优劣。
