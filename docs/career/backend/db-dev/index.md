# 后端开发工程师 · 数据库开发工程师学习路线导读

数据库开发工程师负责数据库相关系统的 **设计、开发、优化与运维支持**，常见工作包括：

- 参与业务系统数据模型设计与评审；
- 负责核心表结构、索引与存储过程等实现；
- 分析线上慢查询与性能瓶颈，给出优化方案；
- 参与数据库分库分表、读写分离、高可用等方案设计；
- 与运维/DBA 协作完成容量规划、监控与应急处理。

## 学习路线模块一览

点击左侧侧边栏或下方链接，可以进入对应模块的详细教程页面：

1. **数据库基础与概念**  
   - 关系模型、表/行/列
   - 主键、外键、唯一约束
   - SQL 基本语法与常见命令  

2. **[数据库原理](/career/backend/db-dev/db-theory/)**  
   - 事务 ACID 特性
   - 并发控制：锁、MVCC、多版本
   - 日志系统：重做日志、回滚日志

3. **[存储引擎](/career/backend/db-dev/storage-engine/)**  
   - 页结构、数据页与索引页
   - Buffer Pool 与刷盘策略
   - 常见存储引擎（如 InnoDB）对上层行为的影响

4. **[SQL 与索引优化](/career/backend/db-dev/sql-advanced/)**  
   - 执行计划解读
   - 索引设计原则
   - 常见慢查询案例分析

5. **[分布式与高可用数据库](/career/backend/db-dev/distributed-db/)**  
   - 主从复制、读写分离
   - 分库分表、中间件
   - 一致性协议与 CAP/Berkeley 视角

6. **[性能调优与排障](/career/backend/db-dev/performance-tuning/)**  
   - 关键监控指标与常见告警
   - 锁等待、死锁分析
   - 热点行/热点表的识别与缓解方案

## 如何使用本路线

- 如果你是 **完全新手**：从「数据库基础与概念」开始，先能读懂常见 SQL，再逐步向内部实现推进。
- 如果你已经是 **后端工程师**：可以直接从「数据库原理」「存储引擎」开始，帮助你把日常问题与底层原理建立联系。
- 如果你在准备 **数据库相关面试**：建议重点关注「SQL 与索引优化」「分布式与高可用」「性能调优与排障」。

> 建议你在学习过程中，结合真实业务场景或开源项目进行练习，例如：分析某个接口在不同索引设计下的执行计划差异。
