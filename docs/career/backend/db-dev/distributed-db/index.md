# 分布式与高可用数据库 · 第 1 章 复制与读写分离

当单机数据库无法满足容量或可用性需求时，你就需要考虑「复制」「读写分离」甚至「分库分表」等方案。

本章先从最经典的 **主从复制 + 读写分离** 讲起。

## 1.1 主从复制的基本模型

以 MySQL 为例，主从复制大致分为：

1. 主库将变更记录到二进制日志（binlog）；
2. 从库拉取 binlog 并写入中继日志（relay log）；
3. 从库重放中继日志，将变更应用到本地数据；
4. 上层通过读写分离中间件，将写请求打到主库、读请求打到从库。

理解这一过程有助于你在延迟、数据不一致、复制错误时快速定位问题。
